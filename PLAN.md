# claude-notify Tauri v2 Implementation Plan

## Architecture Overview

Tauri v2 app with Rust backend (axum HTTP server + focus detection + notifications) and React frontend (settings UI). Runs as a system tray app with no main window.

## Communication Flow

```
Claude Code Hook → curl POST http://localhost:31311/notify (JSON body from stdin)
                 → Rust backend receives request
                 → Checks focus (is terminal foreground?)
                 → If not focused → shows OS native notification
                 → Click notification → activates terminal window
```

Hook config becomes a simple curl call:
```json
{
  "command": "curl -s -X POST http://localhost:31311/notify -H 'Content-Type: application/json' -d @-"
}
```

## Implementation Steps

### Step 1: Scaffold Tauri v2 + React project
- Run `npm create tauri-app@latest` with React + TypeScript template
- Configure as tray-only app (no main window on startup)
- Add dependencies to Cargo.toml

### Step 2: Rust — System tray with menu
- Tray icon with right-click menu: Settings, About, Quit
- Left-click opens settings window
- Tray icon indicates app is running

### Step 3: Rust — Local HTTP server (axum)
- Spawn axum server on `localhost:31311` in background thread
- `POST /notify` endpoint accepts Claude Code hook JSON:
  ```json
  {
    "hook_event_name": "Notification",
    "notification_type": "permission_prompt",
    "cwd": "/path/to/project",
    "message": "Allow command to run?",
    "session_id": "abc123"
  }
  ```
- `GET /health` endpoint for connectivity check
- Parse and validate incoming hook data

### Step 4: Rust — Focus detection (platform-specific)
- **Windows**: Win32 `GetForegroundWindow` + compare with terminal process handles
  - Detect: WindowsTerminal, mintty, ConEmu, cmd, powershell, pwsh, Code (VS Code terminal)
- **macOS**: `NSWorkspace.shared.frontmostApplication` via objc crate
  - Detect: Terminal.app, iTerm2, Alacritty, kitty, Warp, VS Code
- **Linux**: X11 `_NET_ACTIVE_WINDOW` atom via x11rb crate
  - Detect: gnome-terminal, konsole, alacritty, kitty, xterm, VS Code
- Returns bool: is_terminal_focused()

### Step 5: Rust — Notification display
- Use `tauri-plugin-notification` for OS-native notifications
- Title = hook type label ("Permission needed", "Waiting for input", "Task completed")
- Body = project name + message
- On notification click → activate terminal window (Step 6)
- Respect settings (enabled hooks, suppression)

### Step 6: Rust — Terminal window activation (click-to-focus)
- **Windows**: `SetForegroundWindow` (no ShowWindow to preserve snap layout)
- **macOS**: `NSRunningApplication.activate`
- **Linux**: `_NET_ACTIVE_WINDOW` client message via X11

### Step 7: Rust — Settings management
- JSON settings file at platform config dir (e.g., `~/.config/claude-notify/settings.json`)
- Settings schema:
  ```json
  {
    "hooks": {
      "permission_prompt": true,
      "idle_prompt": true,
      "stop": true
    },
    "notification_duration": 5,
    "play_sound": false,
    "auto_start": false,
    "server_port": 31311,
    "suppress_when_focused": true
  }
  ```
- Tauri IPC commands: `get_settings`, `update_settings`, `reset_settings`

### Step 8: React — Settings UI
- Clean, minimal settings page (dark theme matching the notification style)
- Toggle switches for each hook type
- Duration slider
- Sound toggle
- Auto-start toggle
- Port number input
- "Test Notification" button
- Save/reset buttons

### Step 9: Auto-start on login
- Use `tauri-plugin-autostart`
- Toggle from settings UI

### Step 10: Packaging & distribution
- Build for Windows (.msi), macOS (.dmg), Linux (.AppImage/.deb)
- Update README with install instructions
- Add hook config examples for all platforms

## Key Dependencies

### Rust (Cargo.toml)
- `tauri` v2
- `tauri-plugin-notification`
- `tauri-plugin-autostart`
- `tauri-plugin-positioner`
- `axum` — HTTP server
- `tokio` — async runtime
- `serde` / `serde_json` — serialization
- `directories` — platform config paths
- Platform-specific:
  - Windows: `windows` crate (Win32 API)
  - macOS: `objc2` / `cocoa` crate
  - Linux: `x11rb` crate

### Frontend (package.json)
- `react` + `react-dom`
- `@tauri-apps/api`
- `@tauri-apps/plugin-notification`
- `@tauri-apps/plugin-autostart`
